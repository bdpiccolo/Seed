Seed requires input data in a specific format. The data should be in two separate files, one for metadata and the other for abundance data. Seed accepts .csv (comma delimited) data by default, but other delimiting characters may be specified by clicking the 'Show file options' box.

The metaData file should include sample information. Samples should be in rows, with sample names in the first column. Additional columns may include any information about the samples. The taxaData file should include abundance data. Samples should be in rows, with sample names in the first column. Additional columns should list taxa abundance. The abundance may be presence/absence, relative abundance, or count data.

Files that are correctly read by Seed will be shown in the main pannel. Only the first five lines will be displayed. When the taxaData file is loaded, Seed automatically calculates several diversity indices and adds them to the metaData information. These diversity calculations include Shannon, Simpson, and Inverse Simpson indices. They are calculated using the vegan R package and may not be appropriate for every type of data.

Seed includes options for converting abundance data to relative abundance or to presence/absence. Relative abundance is calculated by dividing each taxa count by the sum of all taxa counts for each sample. Presence/absence converts every non-zero taxa to 1. The converted data is used for all visualizations and summaries.

Some plots can be transformed or annotated by metaData features. The features available are all metaData variables and the twenty most abundant taxa. If you need to annotate plots using other taxa, there is an option to include all taxaData variables. This can make things slow for large datasets.

It may be useful to combine some taxa with lowest abundance. There is an option to do this, which sums each taxon which individually represents less than X% of total counts and includes a "combined_other" variable in the new taxaData.

Seed supports saving plots in two different file formats. The user may select either PDF or PNG formats. All saved plots will be in the format selected in the 'Data' tab.
